# Japanese Publication Editorial Review

## Core contract

Produce Japanese that preserves the source while reading as native publication
copy. This is editorial judgment, not an AI-text detector. Never assign an
"AI probability" or rewrite merely to sound different.

The writer cannot self-approve. Final approval requires a fresh reviewer that
has not seen the writer's reasoning or another reviewer's conclusions.

## Inputs

Read the source, target, intended audience, publication family, and maintained
product terminology. If the source is unavailable, report that fidelity cannot
be verified; do not infer missing claims.

## Review passes

Run all passes independently. A factual check elsewhere never waives them.

1. **Frontmatter first** — review title, description, alt text, and CTA labels as
   publication copy, not metadata to waive. Reject literal headline nouns such
   as contextless `方式` or `運営体系` when natural Japanese terms such as `仕組み`
   or `ガバナンス` preserve the meaning better.
2. **Meaning guard** — preserve claims, scope, certainty, conditions, names,
   numbers, dates, links, headings, captions, and CTA intent. Editorial fixes
   must not introduce new meaning.
3. **Native syntax** — flag translation calques and stacked nominal phrases.
   Inspect patterns such as `〜することが可能です`, `〜を通じて`, and
   `〜において`; rewrite only when the sentence is unnatural in context.
4. **Register** — select one document voice and detect mixed `です・ます` and
   `である` endings. Preserve intentional quotations and UI labels.
5. **Flow** — remove redundant subjects, duplicated conclusions, repetitive
   sentence openings, and unsupported promotional filler. Treat repetition as
   evidence, not proof of AI authorship.
6. **Terminology** — use the repository's established QueryPie and technical
   terms consistently. Do not replace precise terms only for variety.
7. **Notation** — check punctuation, half/full-width characters, numerals,
   units, percent signs, spacing, brackets, and list parallelism.
8. **Rendered reading** — check headings, image alt text, figure captions,
   tables, and mobile line flow when browser evidence exists. For video, preserve
   a meaningful title/poster and source-provided captions or transcript; flag
   audio-dependent content that has no equivalent text.

Use `references/review-rubric.md` for severity, deterministic checks, and source
basis.

## Output

Write `japanese-editorial-review.json` with:

```json
{
  "schemaVersion": "global-documentation-sync/v1",
  "artifactType": "japanese-editorial-review",
  "runId": "<run-id>",
  "sourceId": "<source-id>",
  "verdict": "pass|revise",
  "findings": [
    {
      "severity": "critical|major|minor|note",
      "location": "<heading or field>",
      "message": "<specific defect>",
      "suggestion": "<meaning-preserving correction>"
    }
  ]
}
```

Any `critical` or `major` finding means `revise`. Calibrate severity: isolated
wording described as merely `やや`, `少し`, or locally awkward is `minor`, not
`major`. Use `major` only when the defect materially prevents publication or
pervades the document. After correction, rerun this review in a new context. Do
not silently fix or approve unresolved findings.
