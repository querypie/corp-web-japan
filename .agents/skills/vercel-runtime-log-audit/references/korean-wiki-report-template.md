# 한국어 Vercel Runtime Log 보고서 템플릿

이 템플릿은 `corp-web-japan` Vercel Runtime Log 운영 리포트를 한국어로 작성할 때 사용한다.

## 기본 구조

```markdown
# Vercel Runtime Log - {날짜 또는 날짜 범위}

날짜: {YYYY-MM-DD}
감사 수행 시각: {YYYY-MM-DD HH:MM} KST
저장소 위키: `querypie/corp-web-japan`
Vercel 프로젝트: `corp-web-japan`
환경: `production`
실제 로그 확인 범위: {YYYY-MM-DD HH:MM} KST ~ {YYYY-MM-DD HH:MM} KST

## 범위

- 확인한 프로젝트: `corp-web-japan`
- 환경: `production`
- 요청한 KST 기준 로그 창: {요청 창}
- 실제 데이터 범위: {실제 접근 가능 창}
- 확인 초점: 307/308, 404, 5xx, `--level error`

## 요약

- {핵심 결론 1}
- {핵심 결론 2}
- {핵심 결론 3}

## 데이터 무결성 참고

- Vercel Runtime Logs는 현재 환경에서 최근 24시간만 안정적으로 접근 가능하다.
- 이 리포트는 Runtime Log 기반 운영 스냅샷이며 전체 트래픽 집계가 아니다.
- Vercel CLI/API의 limit, dedupe, pagination 제약 때문에 일부 수치는 샘플 또는 capped evidence로 해석해야 한다.
- `--level error` 결과에는 5xx가 아닌 행이 섞일 수 있으므로 5xx 집계는 status field를 기준으로 별도 필터링했다.
```

## 작성 규칙

- 리포트 본문과 섹션 라벨은 한국어로 작성한다.
- Vercel, Runtime Log, HTTP, `production`, status code, 명령어, route path는 그대로 둔다.
- `Scope`, `Executive summary`, `Interpretation`, `Recommended next actions`, `Report generated` 같은 영어 라벨이 남지 않았는지 최종 확인한다.
- 경로나 명령어가 포함된 표 셀은 markdown이 깨지지 않도록 backtick으로 감싼다.
- 수치가 exact total이 아닌 경우 “샘플”, “확인된 행”, “capped evidence”처럼 측정 품질을 명시한다.
