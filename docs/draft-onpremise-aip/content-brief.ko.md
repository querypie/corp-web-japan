# 온프레미스 AIP 웹페이지 기획안

## 1. 콘텐츠 개요

### 작업명

일본 웹사이트용 온프레미스 AI 플랫폼 / 로컬 LLM 어플라이언스 정적 웹페이지

### 콘텐츠 목적

일본 엔터프라이즈와 금융 고객을 대상으로, QueryPie AIP가 단순한
클라우드 LLM 또는 챗봇이 아니라 고객 내부 환경에서 운영 가능한
온프레미스 AI 플랫폼임을 설명합니다.

핵심은 다음 메시지를 전달하는 것입니다.

- 고객 데이터와 업무 지식은 내부에 두고 AI 업무 역량을 높일 수 있습니다.
- 로컬 LLM 단품이 아니라 에이전틱 RAG, AI 에이전트, MCP 실행, 권한,
  승인, 감사, 마스킹이 결합된 운영 플랫폼입니다.
- 금융, 엔터프라이즈, 공공성 높은 업무처럼 보안과 통제가 중요한 환경에서
  실제 업무 실행까지 연결할 수 있습니다.
- PoC에서 시작해 데이터 연결, 모델 튜닝, 에이전트 설계, 통제 체계,
  본 운영 확대까지 단계적으로 도입할 수 있습니다.

## 2. 권장 URL과 구현 가정

### 권장 경로

1차 권장 경로는 `/platforms/aip/on-premise`입니다.

이유는 다음과 같습니다.

- 기존 AIP canonical page인 `/platforms/aip`의 하위 주제로 자연스럽습니다.
- 온프레미스 AI 플랫폼을 별도 솔루션처럼 보이게 하기보다 AIP의 배포/운영
  옵션으로 설명할 수 있습니다.
- 향후 `/platforms/aip/mcp-gateway`, `/platforms/aip/integrations` 같은
  하위 기능 페이지와도 정보 구조를 맞추기 쉽습니다.

대안 경로는 `/solutions/on-premise-ai-platform`입니다. 다만 이 경우 AIP와의
제품 관계가 약해 보일 수 있으므로, 별도 캠페인 또는 산업별 솔루션 페이지가
필요할 때 후순위로 검토합니다.

### 공개 전 상태

콘텐츠 리뷰 전에는 직접 URL 접근 가능한 초안 페이지로 두고, 메뉴와 sitemap
노출은 보류합니다. 제품 전략상 공개가 확정되면 sitemap과 header navigation
반영 여부를 함께 결정합니다.

### 구현 방향

실제 구현 시에는 route-local authoring 원칙을 따릅니다.

- `src/app/platforms/aip/on-premise/page.tsx`가 페이지의 주요 카피와 섹션
  순서를 소유합니다.
- 반복 카드, 다이어그램, CTA 같은 UI 표현은 `src/components/sections/aip/**`
  또는 온프레미스 전용 섹션 컴포넌트로 분리합니다.
- 원본 PDF의 제안서 이미지를 그대로 페이지에 나열하지 않고, 웹페이지용
  메시지 흐름으로 재구성합니다.

## 3. 타깃 독자

### 주요 독자

- 일본 금융기관의 CIO, CISO, DX 추진 책임자
- 은행, 보험, 증권, 카드, 결제, 리스크 관리 조직
- 고객 상담, 여신 심사, 거래 리스크, 내부 감사, 규정 검토 업무를 담당하는
  현업 리더
- 사내 데이터 반출, 클라우드 AI 사용, AI 감사 추적 문제 때문에 생성형 AI
  도입을 보류한 정보시스템/보안 조직
- 온프레미스 또는 프라이빗 환경에서 LLM과 AI 에이전트를 운영해야 하는
  엔터프라이즈 고객

### 독자의 문제

- 범용 클라우드 LLM은 답변 생성에는 유용하지만, 내부 업무 시스템과 승인
  절차, 보안 정책, 감사 로그와 연결되지 않으면 실제 업무 에이전트가 되기
  어렵습니다.
- 금융 데이터, 고객 정보, 내부 문서, 운영 로그를 외부로 반출하기 어렵습니다.
- AI가 업무 시스템을 호출하거나 데이터를 처리할 때 권한, 승인, 마스킹,
  감사 추적을 일관되게 적용해야 합니다.
- 사내 문서 검색을 넘어, 계산, 검증, 문서 작성, 승인 요청, 데이터 처리,
  알림까지 연결되는 업무 실행 환경이 필요합니다.
- PoC는 가능하지만 본 운영으로 확장할 때 보안, 성능, 운영 모니터링,
  컴플라이언스 기준을 충족하기 어렵습니다.

## 4. 핵심 메시지

### 메인 메시지

온프레미스 AI의 핵심은 모델을 내부에 설치하는 것만이 아닙니다. QueryPie AIP는
로컬 LLM, 에이전틱 RAG, AI 에이전트, MCP 실행, ACP 기반 접근 통제와 감사를
하나로 결합해 고객 내부에서 통제 가능한 AI 업무 운영 환경을 제공합니다.

### 보조 메시지

1. 고객 데이터는 내부에 보관하고, AI는 고객 업무에 맞게 최적화합니다.
2. 전용 LLM과 업무별 파인튜닝으로 일본어, 금융 데이터, 고객 도메인 지식을
   반영할 수 있습니다.
3. 에이전틱 RAG는 문서, 규정, 금융 데이터, 승인 절차, 운영 로그를 연결해
   에이전트가 판단하고 실행할 수 있는 맥락을 제공합니다.
4. MCP와 MAC은 AI 에이전트가 호출하는 도구와 시스템 접근을 권한 기반으로
   통제하고 기록합니다.
5. ACP 통제 계층은 권한, 정책, 승인, 감사, 마스킹, UEBA를 통해 기업 통제를
   유지합니다.
6. PoC에서 본 운영까지 업무 선정, 데이터/RAG 연결, 로컬 LLM 튜닝, 에이전트
   설계, 보안 통제, 운영 확대를 단계적으로 검증합니다.

## 5. 콘텐츠 구조

1. Hero: 고객 업무에 최적화된 로컬 LLM과 AI 에이전트를 온프레미스로 제공
2. 문제 제기: LLM 단품과 범용 클라우드 AI만으로는 금융 업무 에이전트를 만들기
   어려움
3. QueryPie의 제안: LLM 어플라이언스와 AI 플랫폼을 고객 내부에 함께 구축
4. 플랫폼 구성: 로컬 LLM, 에이전틱 RAG, AI 에이전트, 스킬, MCP, 접근 통제,
   감사
5. 전용 LLM 역량: H200 GPU 인프라, QLM-1, 일본어/업무 지식 최적화, 도메인
   파인튜닝
6. 에이전틱 RAG와 업무 에이전트: 문서, 규정, 데이터, 승인 절차, 운영 로그를
   업무 실행과 연결
7. 통제 계층: AI Chat, ACP MCP, MAC, UEBA, 권한, 정책, 승인, 감사, 마스킹
8. 금융 업무 유스케이스: 고객 상담, 여신 심사, 거래 리스크, 규정 검토,
   감사 리포트, 내부 지식 검색, 운영 이상 탐지
9. 도입 방식: 업무 선정 -> 데이터/RAG 연결 -> 로컬 LLM 튜닝 -> AI 에이전트와
   MCP 실행 -> ACP 통제와 감사 -> 운영 확대
10. 기대 효과: 내부 데이터 보관, 전용 LLM, 업무 최적화 에이전트, MCP 실행
    통제, 감사 가능성, 금융 보안 대응, 온프레미스 AI 운영
11. CTA: 온프레미스 AIP 상담 또는 데모 요청

## 8. 확인 필요 사항

- `オンプレミスAIプラットフォーム`을 공식 페이지명으로 사용할지,
  `QueryPie AIP for On-Premise AI`를 병기할지 결정해야 합니다.
- QLM-1, H200 GPU 60장 이상, Claude Sonnet 4.6 내부 벤치마크 등 구체 수치를
  공개 웹페이지에 어느 수준까지 노출할지 확인해야 합니다.
- 금융 고객 유스케이스는 실제 고객명 없이 익명화된 업무 시나리오로 표현하는
  것이 안전합니다.
- MAC, UEBA, ACP MCP 같은 제품/기능명은 독자 이해를 위해 첫 등장 시 역할을
  함께 설명해야 합니다.
- 원본 제안서의 시각 자료를 직접 사용하는 대신, 웹페이지용 다이어그램과 카드로
  재구성하는 방향을 권장합니다.
- 공개 확정 시 `/platforms/aip` 본문, header navigation, sitemap, 관련 AIP
  introduction deck 링크와의 연결을 함께 검토해야 합니다.
