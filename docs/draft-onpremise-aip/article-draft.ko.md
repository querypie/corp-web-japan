# 온프레미스 AI 플랫폼

고객 업무에 최적화된 로컬 LLM과 AI 에이전트를 온프레미스에서 제공합니다.

QueryPie AIP는 전용 LLM, 에이전틱 RAG, MCP 실행, ACP 기반 통제를 결합해
일본 엔터프라이즈와 금융기관의 AI 업무 운영을 고객 내부 환경에서 구현합니다.

고객 데이터와 업무 문서는 외부로 내보내지 않고, AI는 고객 업무에 맞게
학습하고 실행합니다. 단순히 답변을 생성하는 LLM이 아니라, 문서와 데이터,
승인 절차, 보안 정책, 감사 로그와 연결되어 실제 업무를 수행할 수 있는
AI 에이전트 플랫폼입니다.

## LLM만으로는 충분하지 않습니다

온프레미스 AI의 핵심은 모델을 내부에 설치하는 것만이 아닙니다.

기업이 원하는 것은 사내에 놓인 모델 하나가 아니라, 업무 지식과 시스템,
권한, 승인, 감사가 결합된 운영 가능한 AI입니다.

로컬 LLM만 도입하면 다음 한계가 남습니다.

- 사내 문서와 업무 데이터에 대한 맥락이 부족합니다.
- 외부 업무 시스템이나 내부 도구와 연결되지 않습니다.
- 누가 어떤 데이터에 접근할 수 있는지 통제하기 어렵습니다.
- AI가 실행한 작업과 판단 근거를 감사 로그로 남기기 어렵습니다.
- 업무별 승인 절차와 보안 정책을 일관되게 적용하기 어렵습니다.

금융기관이나 대기업의 AI 에이전트는 단순한 채팅 도구가 될 수 없습니다.
고객 상담, 여신 심사, 거래 리스크 분석, 규정 검토, 감사 리포트 작성 같은
업무는 내부 데이터, 정책, 승인 절차, 로그와 연결되어야 하며, 실행 결과는
모두 통제되고 기록되어야 합니다.

## QueryPie AIP는 고객 내부에 AI 운영 환경을 구축합니다

QueryPie는 로컬 LLM 어플라이언스와 AI 플랫폼을 고객사 내부에 함께 구축합니다.

이 환경은 고객의 데이터센터 또는 보안 경계 내부에서 동작하며, 업무 데이터,
금융 데이터, 문서, 규정, 내부 시스템을 AI 에이전트가 활용할 수 있는 형태로
연결합니다.

플랫폼은 다음 요소로 구성됩니다.

- 로컬 LLM 어플라이언스
- 온프레미스 AI 플랫폼
- 에이전틱 RAG
- 업무별 AI 에이전트와 스킬
- MCP 기반 도구 실행
- ACP 기반 접근 통제
- 승인과 감사 추적
- 민감정보 마스킹과 이상 행위 탐지

결과적으로 고객은 내부에서 실행되는 통제 가능한 AI를 갖게 됩니다.

## 고객 업무에 맞는 전용 LLM을 만듭니다

QueryPie AI는 플랫폼만 제공하는 것이 아니라, 고객 업무에 맞는 전용 LLM을
직접 개발하고 고도화합니다.

60장 이상의 H200 GPU 인프라와 자체 모델 개발 경험을 바탕으로, 일본어와
업무 지식에 최적화된 QLM-1을 개발하고 있습니다. 여기에 오픈소스 기반
고도화, 내부 벤치마크, 도메인 데이터 파인튜닝을 결합해 고객 환경에 맞는
모델을 구성합니다.

전용 LLM은 다음과 같은 방식으로 확장될 수 있습니다.

- 일본어 이해와 생성 품질 최적화
- 업무 문서, 규정, 지식 기반 학습
- 금융 데이터와 산업별 업무 용어 반영
- Payroll, 금융 데이터, 고객별 업무 도메인에 맞춘 모델 구성
- 고객 환경과 데이터에 맞춘 전용 파인튜닝

중요한 것은 범용 모델을 그대로 쓰는 것이 아닙니다. 고객 업무를 이해하고,
고객 데이터와 정책에 맞게 반응하는 AI 기반을 만드는 것입니다.

## 에이전틱 RAG가 업무 지식을 실행 가능한 맥락으로 연결합니다

AI 에이전트가 정확하게 일하려면, 단순한 문서 검색을 넘어 업무 목적에 맞는
지식 연결이 필요합니다.

QueryPie AIP의 에이전틱 RAG는 업무 문서, 내부 규정, 금융 데이터, 승인 절차,
운영 로그를 연결합니다. 에이전트는 이 맥락을 바탕으로 검색하고, 판단하고,
필요한 도구를 선택하며, 승인 후 실행까지 이어갈 수 있습니다.

연결되는 지식과 데이터는 다음과 같습니다.

- 업무 문서
- 내부 규정
- 금융 데이터
- 승인 절차
- 운영 로그
- 상담 이력과 상품 정보
- 심사 기준과 리스크 정책
- 감사 기준과 통제 항목

에이전트가 사용할 수 있는 업무 흐름은 다음과 같습니다.

- 검색과 조회
- 문서 작성과 요약
- 계산과 검증
- 승인과 결재 요청
- 데이터 처리와 업데이트
- 알림과 커뮤니케이션

이렇게 구성된 RAG는 답변을 찾는 검색 기능에 머무르지 않습니다. 고객 업무
목적에 맞게 AI 에이전트가 판단하고 실행할 수 있는 기반이 됩니다.

## 업무 요구사항에 맞는 전용 AI 에이전트를 설계합니다

QueryPie AIP는 고객의 업무 프로세스를 분석해 전용 AI 에이전트를 설계합니다.

에이전트 설계는 업무 요구사항 분석에서 시작합니다. 업무 목표, 프로세스,
데이터, 제약 조건을 파악하고, 어떤 작업을 AI가 맡을 수 있는지 정의합니다.

그 다음 도메인 LLM, 에이전틱 RAG, 스킬 워크플로우, MCP 도구, 사람 승인,
감사 추적을 결합합니다.

업무 에이전트는 다음처럼 구성됩니다.

- 업무 요구사항 분석
- 도메인 LLM 구성
- 에이전틱 RAG 연결
- 스킬 워크플로우 설계
- MCP 도구 연동
- 사람 승인 절차 반영
- 감사 추적과 운영 모니터링

이를 통해 질의응답, 조사, 심사, 승인, 리포트 작성, 운영 진단 같은 업무를
온프레미스 환경에서 자동화할 수 있습니다.

## AI 생산성은 열고, 기업 통제는 유지합니다

AI를 업무에 연결할수록 통제의 중요성은 더 커집니다.

QueryPie AIP는 AI Chat, ACP MCP, MAC, UEBA를 하나의 통제 계층으로 연결해
권한, 정책, 승인, 감사, 마스킹을 유지합니다.

통제 계층은 다음 역할을 수행합니다.

- 권한: 최소 권한 원칙 기반의 접근 권한 관리
- 정책: 일관된 정책 정의와 자동 적용
- 승인: 민감 작업의 사전 승인과 예외 관리
- 감사: 모든 활동 기록과 지속적 모니터링
- 마스킹: 민감 데이터 자동 식별과 마스킹 적용
- UEBA: 사용자와 에이전트의 행동 분석

AI 에이전트가 사용하는 MCP 도구 호출도 내부에서 통제하고 기록합니다.
MAC은 MCP Access Control로서 조회, 작성, 실행, 승인 같은 도구 권한을
사용자, 역할, 에이전트 단위로 제어합니다.

또한 인증정보 저장소, 응답 마스킹, 요청 감사 기능을 통해 AI가 외부 도구나
내부 시스템을 호출하는 과정까지 관리합니다.

## 금융 업무 에이전트는 로컬 LLM과 통제가 결합될 때 가치가 생깁니다

금융기관의 AI 도입은 일반적인 문서 요약에서 끝나지 않습니다.

고객 상담, 여신 심사, 거래 리스크 분석, 규정 검토, 감사 리포트, 내부 지식
검색, 운영 이상 탐지 같은 업무는 민감한 데이터와 복잡한 정책을 다룹니다.

QueryPie AIP는 이러한 업무를 다음 구조로 연결합니다.

1. 로컬 LLM이 고객 내부 환경에서 동작합니다.
2. 에이전틱 RAG가 상담 이력, 상품 서비스, FAQ, 대출 정책, 심사 기준,
   거래 내역, 규정, 감사 로그, 운영 지표를 연결합니다.
3. ACP 통제가 접근 제어, 데이터 보호, 정책 검증, 위험 관리, 감사 추적을
   적용합니다.
4. AI 에이전트가 상담 응대 초안, 여신 심사 요약, 리스크 리포트, 규정 해석,
   감사 인사이트, 내부 지식 답변, 이상 탐지 알림을 제공합니다.

이 구조를 통해 금융기관은 민감 데이터와 내부 통제를 유지하면서도 AI를 실제
업무 생산성으로 연결할 수 있습니다.

## PoC에서 본 운영까지 단계적으로 확장합니다

QueryPie AIP는 처음부터 전체 업무를 한 번에 바꾸는 방식이 아닙니다.

작게 검증하고, 고객 전용 온프레미스 AI로 단계적으로 확장합니다.

### 1단계: 업무 선정

업무 후보와 우선순위를 정하고, PoC 범위와 목표를 정의합니다.

이 단계에서는 데이터 민감도, 접근 권한, 보안 정책을 함께 검토하고 기대 효과와
성과 기준을 수립합니다.

### 2단계: 데이터와 RAG 연결

대상 데이터 소스를 연결하고 수집합니다. RAG 인덱스와 검색 구성을 만들고,
민감정보 마스킹과 비식별화, 저장 및 전송 암호화를 검토합니다.

검색 정확도와 재현율, RAG 응답 품질을 평가합니다.

### 3단계: 로컬 LLM 튜닝

도메인 데이터로 파인튜닝하고 모델 버전과 실행 환경을 구성합니다.

운영 보안 취약점을 점검하고 프롬프트 인젝션 대응을 검증합니다. 정확도,
응답 품질, 환각 가능성, 레이턴시를 평가합니다.

### 4단계: AI 에이전트와 MCP 실행

AI 에이전트 플로우를 설계하고 MCP 연동과 도구 구성을 진행합니다.

도구 권한, 최소화 검증, 외부 연동 보안 점검, 작업 성공률, 일관성,
에이전트 실행 지연 시간을 확인합니다.

### 5단계: ACP 통제와 감사

ACP 정책과 통제를 설정하고 감사 로그와 리포트를 구성합니다.

정책 준수, 권한 검증, 감사 추적 무결성, 통제 영향도를 분석하고 감사 로그
수집률과 가시성을 확인합니다.

### 6단계: 운영 확대

운영 배포와 모니터링을 시작하고, 확장 계획과 개선 로드맵을 수립합니다.

운영 보안 모니터링, 이상 탐지와 대응 체계를 마련하고 운영 KPI와 지속 개선
성과를 측정합니다.

## 고객 데이터는 내부에 두고, AI 업무 역량은 고객 업무에 맞게 키웁니다

QueryPie AIP는 고객 데이터를 내부에 보관하면서 AI 업무 역량을 고객 업무에
맞게 확장합니다.

기대 효과는 다음과 같습니다.

- 데이터 내부 보관: 모든 데이터가 고객 환경 내부에 저장되어 외부 유출 없이
  안전하게 보호됩니다.
- 고객 전용 LLM: 고객 데이터로 학습하고 최적화한 전용 모델로 정확도와
  보안성을 함께 확보합니다.
- 업무 최적화 에이전트: 업무 프로세스에 맞춘 AI 에이전트가 반복 업무를
  자동화하고 생산성을 높입니다.
- 에이전틱 RAG: 기업 지식과 역할 기반 검색으로 정확하고 신뢰도 높은 답변을
  제공합니다.
- MCP 실행 통제: 외부 도구와 시스템 연동을 정책 기반으로 제어해 안전한 실행
  환경을 보장합니다.
- 감사 가능성: 모든 활동과 결과를 기록하고 추적해 감사와 컴플라이언스 요구에
  대응합니다.
- 금융 보안 대응: 금융권 보안 기준을 충족하는 아키텍처로 민감 데이터와 업무를
  안전하게 보호합니다.
- 온프레미스 AI 운영: 고객 인프라에서 안정적으로 운영하며 확장성과 성능을
  유지합니다.

온프레미스 AI는 모델을 설치하는 프로젝트가 아닙니다. 고객 업무를 이해하고,
데이터와 시스템을 연결하며, 권한과 감사를 유지한 상태에서 AI가 실제 업무를
수행하도록 만드는 운영 체계입니다.

QueryPie AIP는 로컬 LLM, 에이전틱 RAG, MCP 실행, ACP 통제를 하나로 결합해
일본 엔터프라이즈와 금융기관의 안전한 AI 전환을 지원합니다.
