# AS400 / COBOL Migration 콘텐츠 기획안

## 1. 콘텐츠 개요

### 작업명

일본 웹사이트용 IBM i（AS/400）모더나이제이션 정적 웹페이지

### 콘텐츠 목적

일본 기업에 여전히 많이 남아 있는 IBM i（AS/400）와 COBOL/RPG 기반 기간계 시스템을 대상으로, QueryPie AI가 제공할 수 있는 AI 기반 현대화 접근을 소개합니다.

핵심은 다음 문제를 다루는 것입니다.

- AS/400（IBM i）와 COBOL/RPG 유지보수 비용 증가
- COBOL/RPG 엔지니어 고령화와 인력 부족
- 수십 년 누적된 비즈니스 로직의 블랙박스화
- 기존 벤더/SI 의존도 심화
- 클라우드 전환과 DX 지연
- Java, API, Cloud, PostgreSQL, Linux, OCI/AWS 기반 현대화 필요

### 게시 형태 우선순위

1차 게시 형태는 일본 웹사이트의 정적 웹페이지입니다. 공식 주 표기는 `IBM i（AS/400）モダナイゼーション`으로 사용하고, `AS400 / COBOL マイグレーション`은 SEO와 문맥상 보조 표기로 사용합니다.

초기 구현은 `/services/as400-cobol` 직접 URL로 접근 가능한 콘텐츠 초안입니다. 콘텐츠가 리뷰와 수정을 여러 번 거칠 예정이므로, 메뉴와 sitemap에는 아직 노출하지 않고 robots는 `noindex, nofollow`로 둡니다. 한국어 확인용 페이지는 별도 공개 경로를 만들지 않고 `/services/as400-cobol?lang=ko` 쿼리 파라미터로 제공하며, 한국어 카피와 섹션 구성은 `src/app/services/as400-cobol/page.ko.tsx`가 route-local authoring 방식으로 소유합니다.

후순위 확장 콘텐츠는 다음과 같습니다.

| 우선순위 | 형태 | 목적 |
|----------|------|------|
| 1 | 정적 웹페이지 | `/services/as400-cobol` 콘텐츠 초안 구현 |
| 2 | White paper | 리드 수집, 상세 진단 프레임, 세일즈 후속 자료 |
| 3 | Blog | 시장 교육, SEO 유입, 문제 인식 확산 |
| 검토용 | 직접 URL | 메뉴/sitemap 미노출 상태에서 stage 또는 production 확인. 한국어 확인은 `?lang=ko` 사용 |

## 2. 타깃 독자

### 주요 독자

- 일본 중견/대기업의 CIO, CTO, DX 추진 책임자
- 기간계 시스템, Core System, 基幹システム 담당 조직
- AS/400（IBM i）, COBOL, RPG 기반 시스템을 운영하는 정보시스템 부서
- 제조, 유통, 물류, 금융, 보험, BPO, Payroll 영역의 시스템 책임자
- 기존 SI/벤더 유지보수 비용과 전환 리스크를 동시에 고민하는 의사결정자

### 독자의 문제

- AS/400（IBM i）와 COBOL/RPG 시스템이 핵심 업무를 지탱하고 있어 쉽게 건드릴 수 없습니다.
- 시스템은 안정적으로 동작하지만 유지보수 인력과 비용 문제가 커지고 있습니다.
- 수십 년 누적된 업무 로직이 코드 안에 남아 있어 전환 범위와 영향도를 파악하기 어렵습니다.
- 기존 벤더에게 높은 비용을 지불하지만, 구체적인 마이그레이션 로드맵을 잡기 어렵습니다.
- 전체 재개발은 비용과 리스크가 크기 때문에 단계적이고 검증 가능한 전환 방식이 필요합니다.

## 3. 핵심 메시지

### 메인 메시지

IBM i（AS/400）와 COBOL/RPG 기반 기간계 시스템은 일본 기업의 핵심 업무를 지탱해 왔지만, 이제는 유지보수 비용, 인력 부족, 블랙박스화, DX 지연을 해결하기 위한 현대화가 필요합니다. QueryPie AI는 AI를 활용해 기존 COBOL/RPG 로직을 분석하고, Java / API / Cloud / PostgreSQL / Linux / OCI / AWS 기반 전환을 단계적으로 추진할 수 있도록 지원합니다.

### 보조 메시지

1. 레거시 시스템 현대화는 전체 재개발이 아니라, 먼저 현재 시스템을 정확히 분석하는 것에서 시작합니다.
2. AI는 COBOL/RPG, DB2/Oracle, 영향도 분석, 설계서 생성, 테스트 케이스 생성에 활용될 수 있습니다.
3. 분석 결과를 바탕으로 Java 변환, API 생성/연계, PostgreSQL/Linux/OCI/AWS/Cloud 전환을 단계적으로 검토할 수 있습니다.
4. 핵심 업무를 멈추지 않기 위해 진단, PoC, 변환, 구현, 테스트, 운영 안정화까지 이어지는 단계적 진행이 필요합니다.
5. 일본 시장에서는 `2025年の崖`, COBOL 엔지니어 고령화, IBM i 유지보수 비용 증가가 현대화 수요를 키우고 있습니다.

## 7. 확인 필요 사항

- `IBM i（AS/400）モダナイゼーション`을 공식 주 표기로 사용한다. `AS400 / COBOL マイグレーション`은 검색성과 문맥을 위해 보조 표기로 사용한다.
- AI 기반 COBOL/RPG 분석, 설계서 생성, 테스트 케이스 생성, Java/API/Cloud 전환은 충분히 공개 문구로 사용 가능하다. 다만 실제 페이지에서는 기술 목록을 단순 나열하는 데 그치지 않고, “기존 업무 로직을 이해하고 단계적으로 전환 리스크를 낮춘다”는 고객 가치와 연결해 표현해야 한다.
- AS/400（IBM i）직접 연결, DB2/Oracle 분석, Java 변환 등은 모두 QueryPie가 제공하는 AS400 / COBOL Migration 서비스 범위에 포함된다. 현행 시스템 분석, 데이터베이스/스키마 분석, Java 변환 산출물 생성, 전환 구현 지원까지 포괄할 수 있다는 전제로 자신 있게 기술한다.
- Payroll 사례의 구체적인 일정감은 Brant가 이미 시스템 현황을 대략 파악하고 있었던 특정 고객 맥락이다. 일반 홈페이지 공개 문구에는 특정 기간 숫자를 사용하지 않고, 초기 진단과 PoC를 통해 현행 시스템을 이해하고 전환 범위와 로드맵을 구체화한다는 흐름으로 표현한다.
- 상담 CTA의 기본 경로는 `/contact-us`로 확정한다. 버튼 문구는 `相談する`로 두고, 문의 목적이나 제품군을 미리 선택하기 위한 query string 설계는 후순위 과제로 남긴다.
- 정적 웹페이지의 초기 구현 경로는 `/services/as400-cobol`로 확정한다. 콘텐츠 리뷰가 끝나기 전에도 stage/production에 배포될 수 있으므로, 이번 초안에서는 메뉴와 sitemap에 노출하지 않고 robots를 `noindex, nofollow`로 설정한다.
- 15,000개, 4,000~5,000개, 연간 12조 엔 등 수치성 주장은 초안에서 가능한 한 드러내되, 출처 확인은 별도 작업으로 진행한다. 출처 검증 전에는 단정형보다 유보적인 표현을 사용한다.
- Hero 이미지는 `public/services/as400-cobol/` 아래 route-aligned bitmap asset으로 둔다. 본문에는 단계적 Migration 흐름을 다이어그램과 카드/패널 혼합형으로 구현한다.
