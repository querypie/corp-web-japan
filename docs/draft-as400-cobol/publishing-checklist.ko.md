# 게시 전 체크리스트

## 1. 솔루션 메시지 검증

- [x] `AS400 / COBOL Migration`을 QueryPie의 솔루션 또는 서비스로 명시 가능
- [x] 주 표기는 `IBM i（AS/400）モダナイゼーション`으로 사용하고, `AS400 / COBOL マイグレーション`은 SEO와 문맥상 보조 표기로 사용
- [x] AI 기반 COBOL/RPG 분석, 설계서 생성, 테스트 케이스 생성, Java/API/Cloud 전환은 충분히 공개 가능
- [x] DB2 / Oracle 분석, PostgreSQL 이전, Linux/OCI/AWS 이전을 구체적으로 언급
- [x] Payroll 사례의 구체 일정 표현은 일반 홈페이지 공개 문구에서 사용하지 않음
- [x] 진단, PoC, 변환, 구현, 운영 안정화까지 서비스 범위를 자신 있게 명시

## 2. 일본어 품질 검수

- [ ] 일본어 원어민 또는 일본 마케팅 담당자 검수는 최종 게시 전 후속 검수로 진행
- [x] `AS/400`, `IBM i`, `COBOL`, `RPG`, `レガシーシステム`, `基幹システム`, `モダナイゼーション` 표기 기준 설정
- [x] 과도하게 한국식 영업 표현으로 보이는 문장 제거 방향 설정
- [x] 일본 B2B 기술 서비스 페이지 톤에 맞게 차분하고 짧은 문장 중심으로 작성
- [x] `AS400`은 URL, SEO 보조 키워드, 검색 문맥에서 제한적으로 사용하고 본문 기본 표기는 `IBM i（AS/400）` 또는 `AS/400` 우선

## 3. SEO 검수

- [x] H1과 메타 타이틀에 `IBM i（AS/400）モダナイゼーション` 포함
- [x] 메타 타이틀에 `AS400 / COBOL マイグレーション` 보조 키워드 포함
- [x] H2/H3에 `COBOL マイグレーション`, `RPG`, `基幹システム`, `レガシーシステム刷新`, `2025年の崖` 등 관련 키워드 포함
- [x] 메타 타이틀 초안 작성
- [x] 메타 디스크립션 초안 작성. 회사명은 `QueryPie AI`로 표기
- [ ] 일본 웹사이트 내부 관련 페이지 링크는 이번 콘텐츠 초안에서는 추가하지 않고 후속 공개/노출 단계에서 재검토
- [x] 상담/문의/데모 요청 CTA의 기본 경로는 `/contact-us`로 사용

## 4. 게시 위치 결정

- [x] 1차 게시 형태는 일본 웹사이트 정적 웹페이지로 결정
- [x] 정적 웹페이지는 `/services/` 하위 서비스 페이지로 두는 초기 계획 설정
- [x] White paper 확장 여부는 후순위로 검토
- [x] Blog 확장 여부는 후순위로 검토
- [x] `corp-web-japan`의 콘텐츠 구조 확인
- [x] 정적 페이지 구현 방식은 route-local `page.tsx` 중심으로 하고 시각 컴포넌트는 `src/components/sections/as400-cobol/` 아래 분리
- [x] 슬러그는 `/services/as400-cobol`로 결정
- [x] canonical URL은 `https://querypie.ai/services/as400-cobol`로 설정
- [x] 콘텐츠 초안 단계에서는 production/stage 배포 가능성을 열어두되, robots는 `noindex, nofollow`로 설정
- [x] 메뉴 노출과 sitemap 노출은 후순위 작업으로 분리

## 5. 근거와 표현 리스크

- [ ] 일본 IBM i(AS/400) 고객사 약 15,000개 이상 주장의 출처 확인
- [ ] 약 4,000~5,000개사가 COBOL/AS400을 사용한다는 주장의 출처 확인
- [ ] METI `2025年の崖`와 연간 최대 12조 엔 손실 표현의 원문 출처 확인
- [x] 콘텐츠 초안에는 수치를 가능한 한 드러내되, 출처 검증은 별도 후속 작업으로 진행
- [x] 출처 검증 전 수치 표현은 단정형보다 `とされ`, `といわれています`처럼 유보적인 표현 사용
- [x] Payroll, Hitachi 등 특정 고객/벤더명은 공개 페이지에서 사용하지 않기
- [x] 고객명/벤더명/프로젝트명은 익명화하되, AS400, AS/400, IBM i, COBOL, RPG 등 기술 명칭은 명시적으로 노출
- [x] 익명화된 고객 사례와 실제 운영 시나리오는 포함 가능
- [x] "일본은 전 세계에서 유일하게..." 같은 단정 표현은 사용하지 않기
