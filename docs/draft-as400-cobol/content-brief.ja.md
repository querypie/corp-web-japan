# AS400 / COBOL Migration コンテンツ企画書

## 1. コンテンツ概要

### 作業名

日本向けWebサイト用 AS400 / COBOL Migration 静的Webページ

### コンテンツ目的

日本企業に現在も多く残る AS/400（IBM i）と COBOL/RPG ベースの基幹システムを対象に、QueryPie が提供できる AI ベースのモダナイゼーションアプローチを紹介します。

中心テーマは、次の課題です。

- AS/400（IBM i）と COBOL/RPG の保守コスト増加
- COBOL/RPG エンジニアの高齢化と人材不足
- 長年蓄積されたビジネスロジックのブラックボックス化
- 既存ベンダー／SI への依存度の高まり
- クラウド移行とDXの遅れ
- Java、PostgreSQL、Linux、OCI/AWS ベースのモダナイゼーション需要

### 公開形式の優先順位

一次公開形式は、日本向けWebサイトの静的Webページです。QueryPie が提供するソリューションまたはサービス項目の一つとして、`AS400 / COBOL Migration` または `IBM i Modernization` ページを追加する方向を優先します。

後続コンテンツとして、次の形式を検討します。

| 優先順位 | 形式 | 目的 |
|----------|------|------|
| 1 | 静的Webページ | ソリューション／サービスメッセージとして長期掲載 |
| 2 | White paper | リード獲得、詳細診断フレーム、営業フォロー資料 |
| 3 | Blog | 市場教育、SEO流入、課題認知の拡大 |
| レビュー用 | `/t/*` プレビューページ | 公開前の内部レビューと画面確認 |

## 2. 対象読者

### 主な読者

- 日本の中堅・大企業の CIO、CTO、DX推進責任者
- 基幹システム、Core System、基幹システムを担当する組織
- AS/400（IBM i）、COBOL、RPG ベースのシステムを運用する情報システム部門
- 製造、流通、物流、金融、保険、BPO、Payroll 領域のシステム責任者
- 既存SI／ベンダーの保守コストと移行リスクを同時に検討する意思決定者

### 読者の課題

- AS/400（IBM i）と COBOL/RPG システムがコア業務を支えており、簡単には変更できません。
- システムは安定稼働している一方で、保守人材とコストの課題が大きくなっています。
- 長年蓄積された業務ロジックがコード内に残り、移行範囲と影響度を把握しにくくなっています。
- 既存ベンダーに高い費用を支払い続けているものの、具体的なマイグレーションロードマップを描きにくい状況です。
- 全面再開発はコストとリスクが大きいため、段階的で検証可能な移行方式が必要です。

## 3. 主要メッセージ

### メインメッセージ

AS/400（IBM i）と COBOL/RPG ベースの基幹システムは、日本企業のコア業務を支えてきました。しかし今後は、保守コスト、人材不足、ブラックボックス化、DXの遅れを解決するためのモダナイゼーションが必要です。QueryPie は AI を活用して既存の COBOL/RPG ロジックを分析し、Java / PostgreSQL / Linux / Cloud ベースの移行を段階的に進められるよう支援します。

### 補助メッセージ

1. レガシーシステムのモダナイゼーションは、全面再開発ではなく、まず現行システムを正確に理解することから始まります。
2. AI は COBOL/RPG、DB2/Oracle、影響度分析、設計書生成、テストケース生成に活用できます。
3. 分析結果をもとに、Java 変換、API Wrapper 生成、PostgreSQL/Linux/Cloud 移行を段階的に検討できます。
4. コア業務を止めないために、PoC、影響度分析、移行優先順位の設計が必要です。
5. 日本市場では `2025年の崖`、COBOL エンジニアの高齢化、IBM i 保守コストの増加がモダナイゼーション需要を高めています。
